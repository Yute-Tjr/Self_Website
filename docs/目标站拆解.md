# 目标站拆解

目标站：`https://steven.com/`

抓取日期：2026-06-28

## 站点气质

steven.com 是一个产品型 creator economy 生态网站。它不是传统营销长页，而是用极少文字、强视觉资产、压缩字体、大面积留白、全屏过场和菜单导航营造“操作系统”和“生态圆盘”的感觉。

关键词：

- 极简浅灰背景。
- 深色菜单和预加载过场。
- 工业感超窄大写标题。
- Host Grotesk 风格小字号正文。
- 大型 3D 场景、canvas 或 Rive/WebGL 资产。
- 红色手写划线作为少量情绪标记。
- 固定导航、时间显示、菜单抽屉、色彩模式开关。

## 全站页面结构

目标站公开导航包括：

- About Us：首页或主使命页。
- Creator Media：内容页模板一。
- Creator Communities：内容页模板二。
- Creator Products：内容页模板三。
- Creator Tech：内容页模板四。
- Join Us：生态链接页。

对作品集的启发：

- 不做普通“首页、项目、关于、联系”的模板导航。
- 保留参考站的生态导航气质，把个人能力和项目拆成几个“系统模块”。
- 首页作为主叙事入口，菜单作为主要地图。
- 项目页复用目标站内容页模板，每个项目有一个强视觉首屏、一个双语说明区、一个证据/结果区。

## 首页拆解

首屏结构：

- 背景：接近 `rgb(238, 241, 243)` 的浅灰。
- 导航：固定顶部，最大高度约 64px。
- 左上：`STEVEN.COM` 标志，极小、黑色。
- 中上：城市缩写和时间，例如 `( L.A LDN NYC ) 00:42 08:42 03:42`。
- 右上：`MENU`。
- 中央偏右：大型圆环 3D 资产，带内圈、外圈、文字环绕和中心金属球体。
- 左下：超窄大写标题 `THE OPERATING SYSTEM FOR THE CREATOR ECONOMY.`。
- 左下标题下方：`About Us` 文本按钮和圆形箭头按钮。
- 右下：极小色彩模式开关。

首屏可复刻规则：

- 首页必须首屏即建立品牌感，不使用大段自我介绍。
- 标题控制在 3-5 行，使用压缩大写字体。
- 主视觉用 AI/Agent 系统隐喻替换圆环，例如“AI PM operating system”或“AI agent workflow ring”。
- 按钮保持低调，避免商业 SaaS 风格 CTA。

## 菜单拆解

菜单交互：

- 点击 `MENU` 后，右侧出现深色菜单面板。
- 左侧页面被暗色遮罩压暗，原有 3D 场景仍隐约可见。
- 右侧面板宽度约占桌面 35%。
- `MENU` 文案切换到 `CLOSE`。
- 菜单项为巨大的 Field Gothic 大写标题。
- 每项左侧有括号和小圆形选中状态。
- 当前项高亮为近白色，其他项为灰蓝色。
- 底部有版权、Terms、Privacy、Cookies、署名链接。

菜单可复刻规则：

- 菜单是主导航，不另做常规横向导航列表。
- 菜单项用英文，保持目标站气质。
- 中文不进入菜单项，避免破坏字体节奏。
- 菜单打开时应有遮罩、滑入、文字逐项出现或透明度变化。

## 内容页拆解

以 Creator Media 为代表：

加载过程：

- 先出现深色全屏预加载。
- 中心显示 `STEVEN.COM` 标志。
- 标志上方有红色手写划线。
- 底部中心显示数字进度，例如 `100`。
- 完成后转入浅灰页面。

首屏：

- 全幅场景图占据可视区主体。
- 视觉为半写实、半手绘的场景空间，带人物、屏幕、器材、光影。
- 页面底部中心是超压缩大标题，例如 `CREATOR MEDIA`。
- 标题上叠加红色手写划线。
- 左下有小圆形状态/轨道控件。
- 底部中心有小箭头按钮。

滚动后的信息区：

- 巨大正文标题，非压缩字体，局部词语使用红色强调。
- 细线分隔。
- 左侧小标题，例如 `PEOPLE ARE THE PLATFORM`。
- 右侧正文说明，字号中等，行高宽松。

项目或人物网格：

- 3 列图片 masonry。
- 图片大，间距克制。
- 图片下方只有姓名或短标签。
- 不使用传统卡片阴影、标签堆叠或多层文字。

内容页可复刻规则：

- 每个核心项目页需要一个全屏场景视觉。
- 滚动后先给一句巨大判断，再给项目介绍。
- 项目证据以图片、指标、结构图、仓库链接和关键结果呈现。
- 避免传统三栏功能卡片。

## Join Us 页面拆解

结构：

- 浅灰背景。
- 顶部固定导航一致。
- 左侧大标题 `JOIN US`。
- 右侧小说明 `Useful ecosystem links`。
- 下方一条细分隔线。
- 主体为两行大矩形链接区域。
- 每个链接区域内部有一个细描边胶囊按钮。
- 五个链接分别面向 Creators、Founders、Businesses、Investors、A-Players。

对作品集的启发：

- 联系页可改为 `CONNECT` 或 `JOIN THE WORKFLOW`。
- 五个入口可替换为 Resume、GitHub、Email、RAG Project、Research/Graduate Application。
- 保留大矩形和内嵌胶囊按钮的形式。

## 字体与排版

目标站实际读取到：

- 标题字体：`Field Gothic Trimmed`, sans-serif。
- 正文字体：`Host Grotesk`, Arial, sans-serif。
- 首页标题示例：约 60px，行高 60px，字距约 -0.6px。
- 任务页大标题示例：可达 109px 到 208px，字距负值明显。
- 正文：Host Grotesk，16px 到 30px。

当前本地可用字体：

- `Fonts/Field Gothic TEST/WOFF/FieldGothicTEST-No.36.woff` 等多种 Field Gothic TEST 字重/宽度。
- `Fonts/Northwell/*`，可作为手写风格参考，但不是目标站红色手写线的直接字体替代。

字体实现注意：

- Field Gothic TEST 可能存在字符集限制，英文大写标题优先使用。
- 中文正文不能用 Field Gothic，应使用系统中文字体或后续引入可授权中文字体。
- 项目说明双语区中，英文优先匹配 Host Grotesk，中文使用稳定清晰字体。

## 色彩

目标站主要色彩：

- 浅灰背景：接近 `#eef1f3`。
- 主文字：接近 `#2b2b2b`。
- 深色菜单/预加载：接近 `#2b2b2b`。
- 深色模式文字：接近 `#eef1f3`。
- 次级灰蓝文字：接近 `#717c83`。
- 手写强调红：接近珊瑚红或红橙。

作品集色彩策略：

- 以浅灰、深炭灰、灰蓝为主。
- 只保留一个红色强调，用于划线、关键词、加载笔触。
- 不使用蓝紫 AI 渐变。
- 不使用多彩标签。

## 动效清单

需要复刻的关键动效：

1. 预加载过场：深色背景、中心标志、红色划线、底部进度数字。
2. 页面进入：从预加载过渡到浅灰主视觉。
3. 首页主视觉：3D 圆环或 AI 系统环形资产缓慢运动。
4. 菜单打开：右侧面板滑入，背景压暗，菜单项逐项显现。
5. 菜单关闭：反向滑出，页面恢复。
6. 色彩模式切换：右下极小 toggle，浅色/深色切换。
7. 平滑滚动：参考站使用 Lenis 类平滑滚动体验。
8. 内容页滚动：场景图到大字信息区的顺滑转场。
9. 按钮悬停：低调线框、圆形箭头、文字位移或透明度变化。

实现边界：

- 首版可用 CSS/GSAP/Three.js 实现相似动效，不必第一阶段完全还原所有 WebGL 复杂度。
- 如果 3D 资产不能马上完成，先用生成图或 Three.js 简化环形系统作为首版资产，但接口要为后续替换保留。
- 所有动效必须支持 `prefers-reduced-motion`。

## 授权边界与可复用范围

用户已声明获得目标站授权，因此后续复刻时可以直接使用目标网站的源代码、3D 模型、动画和 UI 组件设计，不需要再把目标站版权作为实现阻碍。

可以直接复用或一比一复刻：

- 源代码结构和组件实现。
- 3D 模型、canvas、Rive/WebGL 资产和动画逻辑。
- UI 组件设计、菜单结构、按钮样式、切换动效。
- 信息节奏。
- 排版比例。
- 字体气质。
- 色彩系统。
- 菜单结构。
- 预加载和页面转场逻辑。
- 场景化项目叙事方式。

仍需要替换或重写：

- 目标站品牌名称、业务文案和法律页，因为作品集身份不同。
- Creator economy 原始业务内容，需要替换为个人 AI PM、AI Agent、RAG、YOLO 和申请叙事。
- 无关的城市时间装饰，除非后续确认它服务个人叙事。
