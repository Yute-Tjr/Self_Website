# 技术方案

## 技术栈选择

推荐技术栈：

- 构建工具：Vite。
- 框架：React + TypeScript。
- 路由：React Router。
- 动效：GSAP + ScrollTrigger 用于页面转场、预加载、菜单和滚动叙事；CSS transition 用于基础 hover。
- 3D/场景：Three.js 或 Rive，首版可用 Three.js 简化系统环形视觉，后续替换为更复杂资产。
- 样式：原生 CSS modules 或分层 CSS 文件，避免过度依赖默认组件库。
- 测试：Vitest + Testing Library；关键路由和内容映射写测试。
- 代码质量：ESLint + TypeScript strict。

选择原因：

- 作品集是强视觉交互网站，不需要 Next.js 的服务端能力作为第一优先。
- Vite 启动快，适合小阶段快速验证。
- React Router 能清晰承载 `/`、`/projects/:slug`、`/connect` 等路由。
- GSAP 适合复刻目标站的预加载、菜单和滚动转场。
- Three.js 能承接首页 3D 系统环视觉，避免用纯 CSS 假图。

## 核心模块

### AppShell

职责：

- 放固定导航。
- 放菜单覆盖层。
- 放色彩模式开关。
- 放路由出口。
- 管理页面转场壳层。

验收：

- 每个路由都显示相同导航。
- 菜单打开和关闭不破坏页面滚动状态。
- 右下 toggle 在桌面和移动端都不遮挡主要内容。

### Navigation

职责：

- 左上品牌名。
- 中部可选状态信息。
- 右上菜单按钮。
- 保持单行，不超过 80px 高。

作品集适配：

- 品牌可用 `JIARUI TONG`，但视觉宽度要接近目标站。
- 中部城市时间保留。

### MenuOverlay

职责：

- 右侧深色滑入菜单。
- 背景遮罩。
- 大字导航项。
- 底部链接。

菜单项：

- `ABOUT`
- `RAG AGENT`
- `VALIDATION SKILL`
- `OBB DETECTION`
- `TECH SYSTEM`
- `CONNECT`

### Preloader

职责：

- 进入站点或路由切换时显示深色过场。
- 中心品牌标志。
- 红色手写划线。
- 底部进度数字。

实现规则：

- 进度可以是视觉进度，不伪装真实网络进度。
- 需要 reduced motion fallback：直接淡入页面。
- 不超过可接受等待时间，避免影响可用性。

### HomeScene

职责：

- 首页的核心 3D/视觉资产。
- 用环形系统表达个人 AI 工作流。

首版方案：

- Three.js 创建简化环形结构：外圈、内圈、节点、中心球体。
- 节点代表 Problem、Data、Retrieval、Model、Evaluation、Product。
- 保持浅灰、金属、低饱和视觉，不做彩色科技感。

后续方案：

- 生成或建模更接近 steven.com 的圆环资产。
- 根据滚动或鼠标做轻微转动。

### ProjectPage

职责：

- 按 slug 渲染项目详情页。
- 项目介绍双语。
- 显示系统链路、指标、复盘、链接。

数据来源：

- `src/content/projects.ts` 统一维护。
- 项目字段包括 `title`、`route`、`heroTitle`、`visualType`、`summaryZh`、`summaryEn`、`metrics`、`process`、`reflection`、`links`。

### ConnectPage

职责：

- 复刻 Join Us 页面结构。
- 提供简历、GitHub、邮箱、项目链接。

## 样式系统

### 字体

计划：

- 英文标题：Field Gothic TEST，优先 No.36 或接近目标站宽度的版本。
- 英文正文：若不能合法引入 Host Grotesk，则使用系统 sans 或后续购买/授权字体；先用 CSS 变量预留。
- 中文正文：系统中文字体，保持清晰，不强行套压缩字体。
- 手写线：优先用 SVG path 或生成图片，不依赖 Northwell 写正文。

### 色彩 tokens

```text
--color-paper: #eef1f3;
--color-ink: #2b2b2b;
--color-muted: #717c83;
--color-line: rgba(43, 43, 43, 0.18);
--color-red: #e85f4f;
--color-dark: #2b2b2b;
--color-dark-text: #eef1f3;
```

### 形状

- 页面主体基本无圆角。
- 胶囊按钮使用 full radius。
- 大矩形链接区域使用 0 到 2px radius。
- 不使用大圆角卡片。

## 动效实现边界

首版必须实现：

- 菜单滑入和背景遮罩。
- 首页元素进入动画。
- 按钮 hover 和 active feedback。
- 路由切换预加载或过场。
- reduced motion fallback。

第二阶段后实现：

- 3D 环形视觉缓慢运动。
- 内容页滚动触发转场。
- 图片 masonry 进入动画。
- 色彩模式平滑切换。

不做：

- 自定义鼠标。
- 无意义的多重 marquee。
- 过度视差。
- 每个卡片都有循环动画。

## 测试策略

文档阶段无需测试命令。

实现阶段从第一段代码开始遵守测试优先：

- 内容映射测试：每个项目都有中英文说明、路由 slug、外链和指标。
- 路由测试：`/`、项目页、connect 页可渲染。
- 菜单测试：打开、关闭、链接存在。
- 可访问性检查：按钮有 label，色彩模式 toggle 有可读名称。
- 构建检查：`npm run build`。
- 视觉检查：用浏览器截图检查桌面和移动端。

## 阶段验收命令

后续实现阶段计划使用：

```bash
npm run test
npm run lint
npm run build
```

如果引入 3D 或复杂 canvas，还需要浏览器检查：

- 桌面首屏非空白。
- 移动端不溢出。
- canvas 不遮挡导航和 CTA。
- reduced motion 下不出现强制动画。

## 风险与应对

### 字体授权和字符集

风险：Field Gothic TEST 可能不是正式授权生产字体，也可能字符集有限。

应对：

- 文档和开发阶段先作为本地参考。
- 生产发布前确认授权。
- 中文使用独立字体方案。

### 目标站资源授权状态

状态：用户已声明获得目标站授权。

执行规则：

- 后续复刻时可以直接使用目标网站的源代码、3D 模型、动画和 UI 组件设计。
- 不需要再把目标站版权作为实现阻碍。
- 直接复用的代码和资产需要在开发记录中登记来源、用途和落地位置，方便后续维护。
- 目标站品牌名、业务文案和法律页仍要替换为个人作品集内容。
- 大型 3D、图片和视频资产仍需单独评估仓库体积、加载性能和部署方式。

### 过度追求一比一导致延期

风险：WebGL 场景和复杂转场耗时较高。
