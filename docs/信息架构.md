# 信息架构

## 内容定位

这个作品集不是普通简历网页，而是一个“AI 产品与 AI 系统实践作品集”。它需要同时说服两类读者：

- 招聘侧：AI 产品经理、AI Agent 产品、RAG 应用产品相关岗位。
- 申请侧：海外人工智能研究生项目的招生或导师读者。

因此首页不只说“我是谁”，而要表达“我如何把 AI 项目从问题识别推进到系统实现和效果验证”。

## 双语策略

确认策略：项目介绍双语，其余界面尽量保留英文。

执行规则：

- 导航、按钮、菜单项、页脚和短 UI 文案使用英文。
- 项目页的核心说明提供中文和英文两套内容。
- 中文服务国内 AI PM 招聘场景，强调需求、产品判断、MVP、评测和复盘。
- 英文服务海外申请场景，强调 motivation、method、system design、evaluation、reflection。
- 首页短句可使用英文，避免中英文混排破坏 steven.com 的视觉节奏。

## 首页信息顺序

第一屏：

- 固定顶部导航。
- 中央偏右 AI 系统环形视觉。
- 左下英文主标题。
- 一个低调按钮进入项目生态。
- 右下色彩模式开关。

第二屏或滚动后：

- 大号英文判断句。
- 用红色强调一个关键词，如 `AI systems` 或 `evaluation loops`。
- 小标题和正文说明个人定位。

项目生态区：

- 三个核心项目按目标站内容页入口呈现。
- 不做普通卡片网格。
- 每个项目入口使用大字标题、简短英文描述和一张视觉资产。

联系区：

- 采用 Join Us 页面结构。
- 放简历、GitHub、邮箱、核心项目和申请相关入口。

## 项目页内容模板

每个项目页包含：

1. 全屏视觉首屏。
2. 英文大标题。
3. 红色手写划线。
4. 一句英文主题判断。
5. 中文项目说明。
6. 英文项目说明。
7. 系统架构或流程图。
8. 关键结果和评测指标。
9. 失败案例与复盘。
10. GitHub 或资料链接。
11. 下一项目导航。

## 内容可信度规则

- 数字必须来自简历、项目文档或可验证仓库记录。
- 不编造奖项、实习成果、用户数量、star 数和上线数据。
- GitHub 当前可公开读取的 profile 信号：9 个 repositories，201 contributions in the last year，`Shopping_Guidance_Agent` 公开，`Object_detection_YOLO11_OBB` 显示为私有。
- 私有仓库可以在站内介绍项目，但外链需要显示访问限制或只展示本地说明。
